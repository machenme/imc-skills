# Priority Litmus Test — P0/P1/P2 判定标准

## When

PLAN_SECTIONS 阶段，需要判定每个功能的优先级。

## Goal

给每个功能一个有理有据的 P0/P1/P2 标签，而非凭感觉。

---

## Litmus Test

对每个候选功能依次问：

```
P0 Litmus：缺了这个功能，核心闭环能否跑通？
  → 不能 → P0 (Must Have)
  → 能 → 继续往下问

P1 Litmus：没有这个功能，用户体验是否显著下降？
  → 是 → P1 (Should Have)
  → 否 → 继续往下问

P2 Litmus：这个功能是否"有了更好，没有也行"？
  → 是 → P2 (Nice to Have)
  → 否 → 说明不在当前 Scope（放入"明确不做"）
```

## 判定示例

| 功能 | P0? | P1? | P2? | 结论 |
|------|-----|-----|-----|------|
| 用户注册/登录 | 缺了无法识别用户 → 闭环断裂 | — | — | **P0** |
| 社交分享 | 能跑通核心交易闭环 | 无分享功能用户留存率显著降低 | — | **P1** |
| 深色模式 | 不阻碍闭环 | 体验提升但非显著 | 有了更好 | **P2** |
| AI 智能推荐 | 不阻碍闭环 | 手动浏览也能完成核心任务 | 锦上添花 | **P2** |

## 反模式

- **Everything is P0**：所有功能都标 P0 = 没有优先级。P0 数量应 ≤ 功能总数的 30%
- **P2 不敢写**：P2 不是"不重要"，是"当前阶段不做"。诚实标注 P2 比把所有东西塞进 P0 更有价值
